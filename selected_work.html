Qualcomm Software Engineering Intern

I spent the summer of 2018 working as a part of Qualcomm's 802.11ax WiFi team.
11ax is the next generation of WiFi. It allows for four times the data throughput
and it allows a significant increase in dense WiFi network performance.

Specifically I worked with the WiFi PHY (physical layer) modeling team. The team
builds performance simulation models to emulate the end PHY hardware
for the WiFi modems.

As part of the team, I was in charge of building a framework to format, store,
query, and visualize performance simulation results.

The previous framework would dump simulation results to a pdf document, making it
difficult to analyze trends over time and to reprocess the data at a later time.



The result data was formatted and saved as part of the production level Matlab
simulation. The data was then parsed and pushed into a MongoDB database using a
Python script. The result can then be pulled from the database and plotted
using a Jupyter notebook frontend.

Although the current framework build is for Qualcomm's WiFi products, the
framework is easily applied to the company's 5G modem design pipeline.

Medtronic Software Engineering Intern

I spent the summer of 2016 working as a part of Medtronic's ultrasonic surgical
device software team. The team worked on the firmware for the Sonicision line
of cordless tissue disectors



As part of the team, I helped rebuild the audio feedback drivers, implemented
error checking for over-current and over-voltage, and conducted micellanious
bug fixes as the product finished up the development process and transitioned
to the testing process

I coded exclusively in embedded C++ for a TI Picollo MCU. The code was all
tested on a hardware test harness using standard electrical bench test
equipent (osciliscope, multimeter, signal generator), in addition to
TI's embedded development IDE, Code Composer Studios
